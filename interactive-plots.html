Tasks 1 & 2

Session 11 Check‑in

During this week we explored the world of interactive data visualisation and dashboards. I was particularly struck by three things:

  • The power of Quarto dashboards. The lectures showed how recent advances in the Quarto publishing system allow the creation of fully interactive web pages straight from statistical code. Unlike older R Markdown‑based dashboards, Quarto supports modern layouts, value boxes and responsive designs.
  • Storytelling through exploration. The required readings emphasised that dashboards can change conversations by letting audiences explore data on their own. This interactive element encourages users to form their own insights rather than passively consuming a static chart.
  • Explorable explanations. One of the linked examples demonstrated how interactive documents can embed multiple alternative analyses that readers can switch between. This opens up new ways to communicate uncertainty and context.

Despite these insights, a few things still aren’t entirely clear:

  • Balancing complexity and clarity. The examples we saw ranged from simple charts to fairly elaborate dashboards. It’s still unclear to me how to decide when interactivity adds value versus when it distracts from the story.
  • Hosting interactive content. The instructions mention publishing Quarto websites to Quarto Pub, but I’m curious about alternative hosting options and how to handle access control for sensitive dashboards.
  • Extending dashboards with user inputs. Many real‑world dashboards include filters, sliders or other inputs. While Plotly and Quarto support these features, I’d like to learn more about best practices for adding interactive controls without overwhelming users.
